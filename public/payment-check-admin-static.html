页面需求说明 /payment-check/third-party-payout-reconciliation
需求背景

运营需要在同一页面核对各平台三方代付通道余额、订单成功金额、手续费、押金与差额预警状态，快速发现通道余额异常、押金风险和数据同步差异，并支持按条件筛选、排序、分页查看与权限控制。

需求概述
页面入口左侧菜单“代付对账 / 代付三方对账”。
页面目标按平台和商户维度展示代付三方通道每日对账汇总，辅助运营判断余额差异和预警状态。
数据周期页面按日期查看数据；当日数据支持定时刷新，历史数据原则上保持不变，除非运营录入修正金额。
用户角色
运营人员按平台、商户、日期和预警状态查询对账数据，查看差额和押金风险。
财务人员核对三方商户余额、子平台计算余额、当日修正金额与差值金额。
管理员维护商户数据权限，控制不同角色可查看的商户范围。
筛选条件
平台下拉选择平台，已选平台以标签展示，支持点击标签内 × 清空。
商户名称文本输入，按商户名称模糊匹配。
日期日期控件，选择数据所属日期。
差额预警下拉选择开启、关闭或全部，用于筛选差额预警状态。
预警范围下拉选择大于零、等于零、小于零或全部，用于限定差额预警判断范围。
押金预警下拉选择开启、关闭或全部，用于筛选押金预警状态。
当日零收款商户下拉选择显示或隐藏，用于控制当日无收款商户是否进入列表。
核心功能
查询 / 重置查询按当前筛选条件刷新列表；重置恢复默认筛选项并展示全部数据。
排序带 ↕ 的数值列支持点击排序，首次点击升序，再次点击降序，表头同步展示 ↑ 或 ↓。
分页列表底部展示总条数、每页条数、上一页、下一页、页码和跳页输入；筛选或排序后分页状态同步刷新。
定时刷新点击定时刷新后进入自动刷新状态，再次点击关闭自动刷新。
商户数据权限管理打开权限弹窗，勾选角色可查看的商户范围，保存后提示已保存数量。
计算口径说明
子平台计算商户余额昨日三方支付通道打烊金额 + 当日充值金额 - 当日成功代付订单金额 - 当日成功代付订单手续费 + 当日修正金额。
差值金额三方商户余额 - 子平台计算商户余额，用于判断三方余额与平台账面余额是否一致。
商户余额/押金三方商户余额 / 商户押金 * 100%，用于押金预警状态判断。
差额预警当差值金额超出差额范围预警值时，列表展示预警状态，并可在异常信息页记录异常明细。
列表字段说明
所属平台展示三方代付商户归属的平台名称，用于按平台归类和筛选。
商户名称展示子平台配置的三方商户名称，支持通过筛选条件进行模糊查询。
日期展示当前对账数据所属日期，页面筛选区通过日期控件选择该字段。
当日成功支付订单金额子平台记录的当日三方通道成功代付订单总金额，支持点击表头排序。
当日成功订单手续费当日成功代付订单产生的手续费总额，支持点击表头排序。
昨日三方商户打烊余额昨日 23:59:59 时三方商户通道余额，用于参与子平台计算商户余额。
当日充值金额运营录入或系统同步的当日充值金额，蓝色展示表示可进入明细查看。
子平台计算商户余额按昨日打烊余额、当日充值金额、当日代付订单金额、手续费和修正金额计算出的理论余额。
商户押金三方商户缴纳的押金金额，用于押金预警状态判断。
商户余额/押金三方商户余额相对押金的占比，支持点击表头排序。
三方商户余额三方支付渠道 API 返回的平台余额，展示最近一次同步值。
差值金额三方商户余额与子平台计算商户余额之间的差额，支持点击表头排序。
当日修正金额运营核对后录入的修正金额，用于修正账务差异。
差额范围预警值用于判断差额是否触发预警的阈值范围。
差额预警状态展示当前商户差额预警是否开启，筛选区可按开启、关闭或全部过滤。
押金预警状态展示当前商户押金预警是否开启，筛选区可按开启、关闭或全部过滤。
操作提供当前商户对账记录的详情入口，便于后续查看明细或处理异常。

页面需求说明 /payment-check/payout-reconcile-abnormal
需求时间

2026-06-30

需求背景

现有“代付三方对账”页面用于查看商户每日对账汇总，但当系统发现三方余额、子平台计算余额、差值金额等关键指标异常时，运营和财务需要一个独立列表集中追踪异常消息。新增“代付对账异常信息”菜单，用于展示每一次异常触发记录，支持按平台、商户、时间范围查询，便于定位异常发生时间、异常商户与异常原因。

需求概述
页面入口左侧菜单新增“代付对账异常信息”，点击后在当前后台框架内打开异常信息列表页。
页面目标展示代付对账过程中触发的异常消息明细，包括触发时的对账金额、差值金额、异常描述和时间。
使用场景运营发现汇总页有差额预警后，可进入异常信息页查看具体触发记录；财务可按时间范围回溯历史异常。
异常触发规则
差额异常当“三方商户余额 - 子平台计算商户余额”的结果超出配置的差额范围预警值时，记录一条异常消息。
余额异常当三方商户余额接口返回值为空、失败、为负数或明显低于子平台计算余额时，记录异常消息。
押金风险当商户余额/押金比例触发押金预警逻辑时，可记录异常消息并在描述中标明押金风险。
重复记录同一商户同一分钟内若重复触发同一异常类型，前端列表按后端返回记录展示，不在前端合并。
数据来源
对账汇总数据来源于“代付三方对账”汇总计算结果，包括成功订单金额、手续费、昨日余额、充值金额、修正金额等。
三方余额来源于三方支付通道 API 返回的平台商户余额，取异常触发时的最近一次同步值。
异常描述由后端根据异常类型、平台、商户和差值计算结果生成，用于快速说明触发原因。
时间异常消息生成时间，精确到秒，用于时间范围查询和历史回溯。
业务流程
1. 对账计算系统定时或手动刷新代付三方对账汇总数据。
2. 异常判断根据差额范围、三方余额、押金预警等规则判断是否触发异常。
3. 写入记录触发异常后写入异常信息记录，保存触发时的关键金额快照和异常描述。
4. 页面查询用户进入“代付对账异常信息”页面，按平台、商户名称、时间范围查询异常明细。
筛选条件
平台下拉选择异常记录所属平台；默认不限制平台。
商户名称输入商户名称关键字，支持模糊查询；为空时不限制商户。
时间通过开始时间与结束时间筛选异常消息触发时间；开始和结束均为空时默认查询全部样例数据。
页面操作说明
查询 / 重置查询按筛选条件刷新异常列表；重置清空筛选并恢复默认列表。
异常列表展示异常记录序号、平台、商户、对账金额、差值、异常描述与触发时间。列表横向字段较多，支持横向滚动查看。
分页底部支持页码、上一页、下一页、跳页和每页条数切换；筛选后分页总数同步刷新。
刷新点击表格右上刷新图标，提示异常信息已刷新，用于模拟重新拉取最新异常记录。
列表字段说明
序号异常消息记录的唯一编号。
所属平台异常记录归属平台。
商户名称触发异常的三方代付商户名称。
当日成功代付订单金额异常触发时，子平台记录的当日成功代付订单总金额。
当日成功订单手续费异常触发时，当日成功代付订单产生的手续费总额。
昨日三方商户打烊余额异常触发时参与计算的昨日三方商户余额。
当日充值金额异常触发时的当日充值金额或运营录入金额。
子平台计算商户余额按照对账公式计算出的理论商户余额，用于和三方余额比对。
三方商户余额三方通道接口返回的实际商户余额。
差值金额三方商户余额与子平台计算商户余额之间的差额，是异常判断的核心字段。
当日修正金额运营核对后录入的修正金额，异常记录中保留触发时的金额快照。
差额范围预警值触发异常时使用的预警阈值或范围。
异常消息描述记录异常触发原因或摘要。
时间异常消息生成或触发时间。
注意事项
数据快照异常列表展示的是异常触发时的金额快照，后续汇总数据变化不应覆盖历史异常记录。
历史追溯页面应支持按时间范围查看历史异常，便于运营复盘某一时段的通道异常情况。
权限控制异常记录应遵循商户数据权限，仅展示当前用户有权限查看的商户数据。
